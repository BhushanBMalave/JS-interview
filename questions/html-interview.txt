📌 1️⃣ What is HTML?
HTML (HyperText Markup Language) is the standard markup language used to create and structure web pages.

Example:
<!DOCTYPE html>
<html>
  <head>
    <title>My Page</title>
  </head>
  <body>
    <h1>Welcome</h1>
    <p>This is HTML</p>
  </body>
</html>

Multimedia	                 ❌ Needs Flash	         ✅ Native support
Graphics	                   ❌ Limited	             ✅ Canvas & SVG
Storage	                    Cookies only	            Local & Session Storage
Mobile support	             ❌ Poor	                ✅ Mobile friendly
Performance	                Slower	                  Faster

📌 What is the role of DOCTYPE in HTML?
The DOCTYPE (Document Type Declaration) tells the web browser which version of HTML the page is written in and how to render the page correctly.

🔹 Why DOCTYPE is important?
Defines HTML version
Prevents browser quirks mode
Ensures standard-compliant rendering
Improves cross-browser consistency

🔹 Example of DOCTYPE (HTML5)
<!DOCTYPE html>

📌 This declaration tells the browser:
“This document uses HTML5 standards.”

🔹 What happens without DOCTYPE?
Browser enters Quirks Mode
Page may render differently across browsers
CSS layout issues can occur
Older browser behaviors are used


📌 Difference between <head> and <body> in HTML
(with JS placement – interview-ready explanation)

🔹 <head> vs <body> – Core Difference
<head>	               <body>
Contains metadata	     Contains visible content
Not shown on page	     Shown on the page
Helps browser & SEO	   Helps users
Loads before body	     Rendered after head

🔹 <head> Tag – What goes inside?
The <head> section contains information about the document, not the content itself.

Common elements in <head>:
<head>
  <title>My Page</title>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="HTML tutorial">
  <script src="index.js"></script>
  <link rel="stylesheet" href="style.css">
</head>

Purpose of <head>:
✔ SEO
✔ Browser configuration
✔ Responsive design
✔ Linking CSS & JS

🔹 <body> Tag – What goes inside?
The <body> contains everything visible to the user.

Example:
<body>
  <h1>Welcome</h1>
  <p>This content is visible</p>
  <button>Click Me</button>
</body>

🔹 Key Difference in One Line (Interview)
The <head> contains metadata and resource links, while the <body> contains the visible content of the webpage.

📌 Where to place JavaScript link reference?
There are three correct ways, but one is recommended.

✅ 1️⃣ Best Practice – JS at the end of <body> (Recommended)
<body>
  <h1>Hello</h1>
  <script src="script.js"></script>
</body>

Why best?
✔ HTML loads first
✔ Faster page rendering
✔ No DOM errors

📌 Most common interview answer
✅ 2️⃣ JS in <head> using defer (Modern & Recommended)
<head>
  <script src="script.js" defer></script>
</head>

How defer works:
Downloads JS in parallel

Executes after HTML is parsed
✔ Clean
✔ No blocking

⚠️ 3️⃣ JS in <head> without defer (Not recommended)
<head>
  <script src="script.js"></script>
</head>

❌ Blocks HTML parsing
❌ Slower page load

🔹 defer vs async (Interview Bonus)
defer	                            async
Executes after HTML parsing	      Executes immediately
Maintains order	                  Order not guaranteed
Best for DOM scripts	             Best for analytics

📌 Interview-Perfect Answer (Short)
Difference between head and body:
The head contains metadata and resource links, while the body contains visible content.

Where to place JS?
JavaScript should be placed at the end of the body or in the head using the defer attribute for better performance.

🔥 Common Interview Trap
❓ Can JS be placed in both head and body?
✔ Yes, but placement affects performance.



📌 What is the <title> tag in HTML?
The <title> tag defines the title of a web page.
It is displayed in:
Browser tab
Bookmarks
Search engine results (SEO)

📌 The <title> tag is placed inside the <head> section.
✅ Example
<head>
  <title>HTML Interview Questions | Edureka</title>
</head>

🔹 Where does the title appear?
Browser tab
Search engine results page (SERP)
Bookmarks / history

📌 4 Advantages of the <title> Tag
1️⃣ Improves SEO (Search Engine Optimization)
Search engines use the title tag to understand page content.
It is a ranking factor.
📌 A well-written title improves visibility on Google.

2️⃣ Increases Click-Through Rate (CTR)
The title is the main clickable text in search results.
A clear title attracts more users.

✅ Example:

<title>Top 30 HTML CSS Interview Questions – Explained</title>

3️⃣ Better User Experience
Users can easily identify the page from browser tabs.
Helpful when multiple tabs are open.
📌 Without a title, browser shows Untitled Document.

4️⃣ Helps in Bookmarking & History
The title is used as the default name when bookmarking a page.
Makes saved pages easy to recognize later.

🔹 Important Rules of <title> Tag (Interview Points)
✔ Only one <title> per page
✔ Must be inside <head>
✔ Should be 50–60 characters
✔ Should be unique for each page

📌 One-Line Interview Answer
The title tag defines the page title shown in browser tabs and search results, and it improves SEO, user experience, and click-through rates.


📌 HTML <meta> Tag – Detailed Notes
1️⃣ What is the <meta> tag?
The <meta> tag provides metadata about an HTML document.

📌 Metadata = data about data
It is not visible on the webpage but is used by:
Browsers
Search engines
Screen readers
Social media platforms

2️⃣ Where is the <meta> tag used?
Meta tags are always placed inside the <head> section.
<head>
  <meta charset="UTF-8">
</head>

3️⃣ Why are meta tags important?
✔ SEO (search engine optimization)
✔ Responsive design
✔ Character encoding
✔ Browser compatibility
✔ Social media sharing

4️⃣ Commonly Used Meta Tags (With Examples)
🔹 1. charset Meta Tag
Defines character encoding of the webpage.
<meta charset="UTF-8">
Why UTF-8?
Supports all languages
Avoids text corruption

📌 Interview point:
Always place it first inside <head>.

🔹 2. viewport Meta Tag (VERY IMPORTANT)
Used for responsive design.
<meta name="viewport" content="width=device-width, initial-scale=1.0">

Explanation:
Property	Meaning
width=device-width	Match screen width
initial-scale=1.0	Zoom level

📌 Without this, mobile pages look zoomed out.

🔹 3. description Meta Tag
Used by search engines to display page summary.
<meta name="description" content="Learn HTML meta tags with examples">
✔ Improves click-through rate
❌ Does not directly affect ranking

🔹 4. keywords Meta Tag (Deprecated)
Previously used for SEO.
<meta name="keywords" content="HTML, CSS, JavaScript">
📌 Modern browsers & Google ignore it.

🔹 5. author Meta Tag
Defines the author of the document.
<meta name="author" content="Bhushan Malave">

🔹 6. robots Meta Tag
Controls search engine crawling behavior.
<meta name="robots" content="index, follow">

Common values:
Value	Meaning
index	Allow indexing
noindex	Block indexing
follow	Follow links
nofollow	Don’t follow links

📌 Example:
<meta name="robots" content="noindex, nofollow">

🔹 7. http-equiv Meta Tag
Used to simulate HTTP headers.
Example – Refresh Page:
<meta http-equiv="refresh" content="5">

🔁 Refreshes page every 5 seconds

Example – Redirect:
<meta http-equiv="refresh" content="5; url=https://example.com">

🔹 8. Content-Type (Old way – not recommended)
<meta http-equiv="Content-Type" content="text/html; charset=UTF-8">
✔ Replaced by:
<meta charset="UTF-8">

🔹 9. Compatibility Meta Tag (IE)
<meta http-equiv="X-UA-Compatible" content="IE=edge">
📌 Forces latest rendering engine in Internet Explorer.

🔹 10. Open Graph Meta Tags (Social Media)
Used for Facebook, LinkedIn, WhatsApp previews.

<meta property="og:title" content="HTML Meta Tags">
<meta property="og:description" content="Complete guide to meta tags">
<meta property="og:image" content="image.png">
<meta property="og:url" content="https://example.com">

🔹 11. Twitter Card Meta Tags
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="Meta Tags Explained">

5️⃣ Full Example of Meta Tags in <head>
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="HTML Meta Tag Tutorial">
  <meta name="author" content="Bhushan Malave">
  <meta name="robots" content="index, follow">
  <title>Meta Tags</title>
</head>

6️⃣ Meta Tag Interview Questions (Most Asked)
🔸 What is meta tag?

Metadata about HTML document.
🔸 Is meta tag visible?
❌ No

🔸 Where is meta tag placed?
Inside <head>

🔸 Which meta tag is important for mobile?
viewport

🔸 Does meta description affect SEO ranking?
❌ No (but improves CTR)

🔸 Can multiple meta tags be used?
✔ Yes

7️⃣ Best Practices
✔ Use UTF-8 charset
✔ Always add viewport
✔ Write meaningful descriptions
✔ Avoid deprecated keywords tag
✔ Keep description under 160 characters

8️⃣ One-Line Interview Answer (Quick)

Q1. What are Root, Parent, Child & Nested Elements?
🔹 Root Element
The root element is the top-most element of an HTML document.
<html>
  ...
</html>
📌 <html> is the root element.

🔹 Parent Element
An element that contains another element.
<body>
  <p>Hello</p>
</body>
➡ <body> is the parent of <p>.

🔹 Child Element
An element inside another element.
<p>Hello</p>
➡ <p> is a child of <body>.

🔹 Nested Elements
Elements placed inside other elements.
<div>
  <p>
    <span>Text</span>
  </p>
</div>
➡ <span> is nested inside <p>, which is nested inside <div>.

📌 Interview line:
HTML elements can be nested to form a document tree structure.

Q1. What are HTML Attributes? What are the Types of HTML Attributes?
🔹 What are HTML Attributes?
HTML attributes provide additional information about an element.
They are written in the opening tag as name="value" pairs.

Example:
<img src="logo.png" alt="Company Logo">

🔹 Types of HTML Attributes
1️⃣ Global Attributes
Can be used on any HTML element.

Examples:
id
class
style
title
hidden
data-*

2️⃣ Element-Specific Attributes
Work only on certain elements.

Examples:
href   → <a>
src    → <img>
type   → <input>
alt    → <img>

3️⃣ Boolean Attributes
Do not need a value.

Examples:
<input disabled>
<input checked>
<input required>

4️⃣ Event Attributes
Used to handle events (not recommended inline).

Examples:
<button onclick="submitForm()">Submit</button>

📌 Interview line:
Attributes add extra information or behavior to HTML elements.

Q2. What are the id, style & class attributes? When to use what?
🔹 id Attribute
Unique identifier
Used for JavaScript & CSS targeting

<div id="header"></div>

✔ Only one element
✔ Used when uniqueness is required

🔹 class Attribute
Used to style multiple elements

Reusable
<p class="text-red">Text</p>

✔ Preferred for CSS
✔ Multiple elements allowed

🔹 style Attribute
Inline CSS
Applies to single element only

<p style="color:red;">Text</p>

❌ Not recommended
✔ Overrides CSS rules

🔹 When to use what?
Attribute	When to Use
id	Unique element, JS
class	Reusable styles
style	Quick override (avoid)

Q3. What will happen if two elements have the same id?
❌ Invalid HTML
<div id="box"></div>
<p id="box"></p>

Problems:
Breaks HTML standards
JavaScript selects only first element
CSS becomes unpredictable

📌 Interview answer:
IDs must be unique; duplicate IDs can cause DOM and styling issues.

Q4. How to specify Multiple Classes for a single element? What is the style precedence?
🔹 Multiple Classes Example
<div class="card highlight shadow"></div>

✔ Separate classes with space

🔹 Style Precedence (CSS Priority)
Highest → Lowest
Inline styles
ID selector
Class selector
Element selector
Browser default

Example:
#box { color: red; }
.box { color: blue; }
div { color: green; }

➡ Result: Red (ID wins)

📌 !important overrides all (avoid using).

Q5. What are Data Attributes in HTML?
🔹 Data Attributes (data-*)
Used to store custom data on HTML elements.

Syntax:

🔹 Advantages:

✔ Clean HTML
✔ Useful in React / JS apps

📌 Interview answer:
Data attributes allow storing custom data in HTML elements without affecting structure or styling.


Q1. What are the 5 Types of Links in HTML?
HTML links are created using the <a> (anchor) tag.

🔹 1️⃣ External Links
Links to another website.

<a href="https://www.google.com">Google</a>

🔹 2️⃣ Internal Links
Links to another page within the same website.

<a href="about.html">About Us</a>

🔹 3️⃣ Bookmark / Anchor Links
Links to a specific section of the same page.

<a href="#contact">Go to Contact</a>

🔹 4️⃣ Email Links
Opens default email client.

<a href="mailto:info@example.com">Email Us</a>

🔹 5️⃣ Phone Links
Used mostly on mobile devices.

<a href="tel:+919876543210">Call Us</a>

📌 Interview line:
HTML supports external, internal, bookmark, email, and phone links.

Q2. What is the difference between Absolute & Relative URLs?
Absolute URL	      Relative URL
Full path	         Partial path
Includes domain	   No domain
Works everywhere	  Depends on file location

Examples:
Absolute URL

Relative URL

Absolute URLs specify the complete address, while relative URLs are based on the current file location.

<td>	Data cell

✅ Advantages of Tables

✔ Easy to read data
✔ Structured rows & columns

❌ Disadvantages of Tables
❌ Not responsive by default
❌ Not for page layout
❌ Complex styling

📌 Interview note:
Tables should be used only for data, not layouts.

Q4. What is the colspan attribute in HTML?
colspan allows a table cell to span multiple columns.

  <tr>
    <th colspan="2">User Info</th>
  </tr>
  <tr>
    <td>Name</td>
    <td>Age</td>
  </tr>
</table>

📌 Interview answer:
colspan merges two or more columns into one cell.

</style>

Benefits:
✔ Clean HTML
✔ Better control
✔ Responsive-friendly

📌 Interview point:
CSS is the best and modern way to style table borders.

Q2. What is Inline Style in CSS? When to use it in real applications?
📌 Definition:
Inline CSS is written inside the HTML element using the style attribute.

Example:
<p style="color: red; font-size: 18px;">
  Hello CSS
</p>

✅ When to Use:

✔ Dynamic styles via JavaScript
✔ One-off UI changes

❌ Avoid When:

❌ Reusable components
❌ Maintainability is required

📌 Interview Tip:
Inline CSS has the highest priority.

Q3. What is Internal Stylesheet in CSS? When to use it?
📌 Definition:
CSS written inside <style> tag within <head>.

Example:
<head>
  <style>
    p {
      color: blue;
    }
  </style>
</head>

✅ When to Use:
✔ Small web pages
✔ Single-page projects
✔ Prototyping

❌ Avoid When:
❌ Multiple pages
❌ Reusability needed

Q4. What is External Stylesheet in CSS? When to use it?
📌 Definition:
CSS written in a separate .css file and linked to HTML.

Example:
<link rel="stylesheet" href="styles.css">

/* styles.css */
p {
  color: green;
}

✅ When to Use:
✔ Large projects
✔ Multi-page websites
✔ Scalable applications

📌 Most recommended method

Q5. What are the 5 advantages of External Stylesheet?
✅ Advantages:

2️⃣ Reusable across pages
3️⃣ Better performance (browser caching)
4️⃣ Easy maintenance
5️⃣ Cleaner HTML

📌 Interview Line:
External CSS improves maintainability, performance, and scalability.

🎯 CSS SELECTORS – Interview Q&A
Q1. What are CSS Selectors?
CSS selectors are used to select HTML elements and apply styles to them.

p {
  color: blue;
}

Q2. What are the different types of CSS selectors?
🔹 1. Universal Selector
* {
  margin: 0;
}

🔹 2. Element (Tag) Selector
h1 {
  color: red;
}

🔹 3. Class Selector
.title {
  font-size: 20px;
}

🔹 4. ID Selector
#header {
  background: black;
}

📌 Note: ID must be unique.

🔹 5. Group Selector
h1, h2, p {
  color: green;
}

🔹 6. Attribute Selector
input[type="text"] {
  border: 1px solid blue;
}

🔹 7. Descendant Selector
div p {
  color: red;
}

🔹 8. Child Selector
div > p {
  color: blue;
}

Q3. Difference between Class and ID selector?
Class	     ID
Reusable	  Unique
. prefix	  # prefix
Lower specificity	Higher specificity

Q4. What is CSS Specificity?
Specificity decides which CSS rule is applied when multiple rules target the same element.

Priority Order:
Inline > ID > Class > Element

Q6. What is the difference between display properties?

Value	       Description

fixed	       Relative to viewport

Q8. Difference between absolute and relative positioning?
Relative	                     Absolute
Moves relative to itself	     Moves relative to parent
Keeps space	                  Removed from document flow

Q9. What is Flexbox?
Flexbox is a 1-dimensional layout system for aligning elements.

.container {
  display: flex;
  justify-content: center;
  align-items: center;
}

Q10. What is CSS Grid?
Grid is a 2-dimensional layout system.

.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
}

Q11. Difference between Flexbox and Grid?
Flexbox	Grid
1D layout	2D layout
Row OR column	Rows AND columns

📱 RESPONSIVE DESIGN – Interview Q&A
Q12. What is Responsive Web Design?
Responsive design makes websites adapt to different screen sizes.

Q13. What is a Media Query?
Media queries apply CSS based on screen size.

@media (max-width: 768px) {
  body {
    background: lightgray;
  }
}

Q14. What is the viewport meta tag?
Controls page scaling on mobile devices.
<meta name="viewport" content="width=device-width, initial-scale=1.0">

Q15. What are CSS units used in responsive design?
Unit	   Use
%	      Relative
em/rem	 Font-based
vw/vh	  Viewport
px	     Fixed

Q16. Difference between em and rem?
em	rem
Relative to parent	Relative to root

Q17. What is Mobile-First Design?
Designing for small screens first, then scaling up.

/* Mobile */
body { font-size: 14px; }

/* Desktop */
@media (min-width: 768px) {
  body { font-size: 16px; }
}

Q18. What are breakpoints?
Breakpoints define screen widths where layout changes.
Common breakpoints:
576px
768px
992px
1200px

🔥 Quick Interview Tips
Use Flexbox for components
Use Grid for layouts
Prefer external CSS
Avoid inline styles
Always use viewport meta tag
